# 📝 Blogging Platform

A full-stack **MERN Blogging Platform** with authentication, role-based access, and an admin panel.
Users can create, read, update, and delete blog posts, while admins manage users and content.

---

## 🚀 Features

- 🔐 JWT-based Authentication (signup, login, logout)
- 📝 Blog CRUD (create, read, update, delete)
- 💬 Comments & Likes
- 👤 User Profiles with Follow system
- 🛡️ Admin Dashboard (manage users, posts, roles)
- 📂 Image Uploads via Multer
- ⚡ RESTful API Architecture

---

## ⚙️ Local Setup

### Prerequisites
- Node.js v18+
- MongoDB (local or [MongoDB Atlas](https://www.mongodb.com/cloud/atlas))

### 1) Backend

```bash
cd backend
cp .env.example .env   # edit with your MongoDB URI & secrets
npm install
npm run dev
```

Seed an admin user (optional):
```bash
npm run seed:admin
```
Default admin creds: `admin@example.com` / `admin12345`

### 2) Frontend

```bash
cd frontend
cp .env.example .env   # edit VITE_API_URL if needed
npm install
npm run dev
```

Open the app at `http://localhost:5173`.

## 🌍 Deployment

### Backend → Render

1. Push this repo to GitHub.
2. Go to [Render Dashboard](https://dashboard.render.com) → **New** → **Blueprint** → select your repo.
   - Render reads `render.yaml` and auto-creates the service.
3. Set these **Environment Variables** in Render:
   | Variable      | Value                                      |
   | ------------- | ------------------------------------------ |
   | `MONGODB_URI` | Your MongoDB Atlas connection string        |
   | `JWT_SECRET`  | (auto-generated by blueprint)              |
   | `CORS_ORIGIN` | Your Vercel URL, e.g. `https://your-app.vercel.app` |
4. After deploy, open Render **Shell** and run:
   ```bash
   npm run seed:admin
   ```

### Frontend → Vercel

1. Go to [Vercel Dashboard](https://vercel.com/dashboard) → **Add New Project** → import your GitHub repo.
2. Set **Root Directory** to `frontend`.
3. Add this **Environment Variable**:
   | Variable       | Value                                        |
   | -------------- | -------------------------------------------- |
   | `VITE_API_URL` | `https://<your-render-service>.onrender.com/api` |
4. Click **Deploy**.

## 🔑 API Endpoints

### Auth
| Method | Endpoint          | Description   |
| ------ | ----------------- | ------------- |
| POST   | `/api/auth/register` | Register   |
| POST   | `/api/auth/login`    | Login      |
| POST   | `/api/auth/logout`   | Logout     |
| GET    | `/api/auth/me`       | Current user |

### Posts
| Method | Endpoint             | Description              |
| ------ | -------------------- | ------------------------ |
| GET    | `/api/posts`         | List posts (paginated)   |
| GET    | `/api/posts/:slug`   | Single post by slug      |
| POST   | `/api/posts`         | Create post (auth)       |
| PUT    | `/api/posts/:id`     | Update post (auth)       |
| DELETE | `/api/posts/:id`     | Delete post (auth)       |

### Users
| Method | Endpoint               | Description              |
| ------ | ---------------------- | ------------------------ |
| GET    | `/api/users`           | List users (admin)       |
| GET    | `/api/users/:id`       | Get user profile         |
| PATCH  | `/api/users/:id/role`  | Change role (admin)      |
| DELETE | `/api/users/:id`       | Delete user (admin)      |
